Introducción
El desarrollo frontend es esencial para materializar la experiencia de usuario definida en el proyecto Censo Rural, ya que transforma requisitos y flujos de interacción en interfaces navegables, accesibles y consistentes. En Extreme Programming (XP), los ciclos cortos y la retroalimentación continua favorecen entregas frecuentes de módulos funcionales independientes (por ejemplo, una navbar, tarjetas de contenido o formularios), lo que facilita validar decisiones de diseño con usuarios reales y ajustar rápidamente antes de invertir en desarrollo de mayor costo. En este tutorial, se presenta una página web didáctica que demuestra cómo estructurar HTML semántico, aplicar CSS con variables del sistema de diseño del proyecto y agregar JavaScript básico para la interactividad, priorizando usabilidad y accesibilidad.
Selección del tema del tutorial
Para mantener coherencia con el dominio, el tutorial elegido es:
“Guía de capacitación digital Censo Rural: componentes base (Navbar, Cards y Paso a paso)”.
Este tutorial servirá como referencia rápida para encuestadores y personal de apoyo, mostrando patrones reutilizables que se integrarán en la aplicación de captura y en el repositorio de información.

III. Diseño y estilos CSS (basado en code.txt)

Criterios: Variables del sistema de diseño, componentes reutilizables y lineamientos de accesibilidad.

1. Variables del sistema de diseño
Definir en :root:

--primary: #60a5fa (Azul de acción)
--bg: #0b0f17 (Fondo oscuro para legibilidad)
--surface: #0f172a (Contenedores)
--radius: 12px (Bordes redondeados)
Tipografía base ≥ 16 px y escala fluida para mejorar lectura


[Espacio para imagen: “Inspección de variables en DevTools”]

2. Componentes reutilizables


Navbar con desenfoque

Contenedor con fondo semitransparente sobre --bg y backdrop-filter: blur(8px) (con fallback de color sólido).
Estados :hover, [aria-current="page"] y foco :focus-visible.



Cards (.card)

Fondo --surface, radio var(--radius), sombra suave y padding consistente.
Admite título, descripción y CTA.



Estilos de lista (.steps)

<ol> sin bullets por defecto; usa contadores CSS para círculos numerados.
Espaciado vertical amplio y contraste suficiente entre número y texto.




[Espacio para imagen: “Navbar + Cards + Lista .steps”]

3. Accesibilidad y usabilidad

Contraste mínimo 4.5:1 para texto normal; 3:1 para texto grande y elementos activos.
Tamaño de fuente mínimo 16 px y altura de línea confortable (≥ 1.5).
Foco visible, zonas táctiles generosas y etiquetas/roles adecuados.


[Espacio para imagen: “Chequeo de contraste y foco visible”]


IV. Funcionalidad básica con JavaScript
1. Interactividad simple

Modo oscuro: toggle que alterna una clase en <html> (p. ej., data-theme="dark"), almacena preferencia en localStorage y respeta prefers-color-scheme.
Navegación y feedback:

Enlaces del índice hacen scroll suave.
Al finalizar una sección (botón “Marcar como visto”), mostrar confirmación no intrusiva (p. ej., toast o aria-live="polite").




[Espacio para imagen: “Toast de confirmación”]

2. Consideraciones de calidad

No bloquear la UI; degradar elegantemente si JS no carga.
Evitar dependencias pesadas para mantener tiempos de carga bajos.
Validar accesibilidad: navegación por teclado y compatibilidad con lectores de pantalla.


V. Conclusiones y entregables
Conclusiones
La maquetación programática con HTML semántico, CSS con variables del sistema de diseño y JavaScript ligero promueve consistencia, mantenibilidad y escalabilidad en el proyecto Censo Rural. Definir componentes reutilizables (Navbar, Cards, Listas de pasos) y pautas de accesibilidad (contraste, foco visible, tipografía legible) facilita entregas frecuentes y validación temprana en línea con XP, reduciendo retrabajo e incrementando la calidad percibida por los usuarios finales.
